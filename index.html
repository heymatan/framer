Device Frames

A set of reusable HTML/CSS frames for context of webpages, with a fallback when no page is displayed.

    Tenets:
  • All device sizes are declared in px. This is to ensure that the iframe sizing is correct even when zoomed (ems would resize the container).
  • The outermost container for a frame is a <div class="device-frame DEVICE-NAME"> where DEVICE-NAME is a unique identifier.
  • The <div class="device-frame"> should be set to the size of the desired viewport, with sufficient padding to contain all device chrome and decoration.
  • The device chrome and decoration should be position: absolute so as not to interfere with the device-frame sizing. This way, the developer does not have to calculate the total size of the device in pixels; they simply include the appropriate HTML, give the outer frame the display size they want, and let the styles handle the presentation.
  • All content should go into a nested <div class="viewport">

Desktop Browser (Proportion Resizeable)

This is a desktop browser frame. It is able to be resized and reproportioned. This demo is built with pure HTML/CSS, which is the preferred way to build frames, as it is the lightest in filesize.

Ipad (Proportion Locked)

This is an ipad chrome. Its screen is a fixed proportion. This demo is also built with pure HTML/CSS.

Kindle (Proportion Locked)

This is kindle chrome. This demo is using an image for the frame, as opposed to one built completely in CSS. This would allow for faster iteration if there were a new device but not enough time to build it in code. It still offers significant file size savings over the current model, because the chrome is not being included in every image, and we can take advantage of .jpg compression for the screenshots since transparency is not necessary without the drop shadow.

  • Resolution with UI: 963x573
  • Next steps: Add landscape class to handle other orientation
  • Next steps: Rebuild in HTML/CSS
